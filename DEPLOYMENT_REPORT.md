# Раздел для отчёта: Развёртывание проекта и настройка CI/CD

## Размещение проекта в GitHub

Исходный код веб-приложения был размещён в удалённом репозитории GitHub. Репозиторий используется для хранения исходных файлов проекта, отслеживания изменений и организации автоматического развёртывания приложения.

В репозиторий были добавлены основные файлы проекта: исходный код React-приложения, файл `package.json`, конфигурационные файлы для деплоя и файл `.gitignore`, исключающий загрузку служебных папок `node_modules` и `dist`.

## Подготовка проекта к деплою

Проект разработан на React с использованием сборщика Vite. Для запуска проекта локально используется команда:

```bash
npm run dev
```

Для подготовки production-версии используется команда:

```bash
npm run build
```

После выполнения команды сборки создаётся папка `dist`, в которой находятся готовые статические файлы сайта. Именно эта папка публикуется на хостинге.

## Настройка CI/CD

Для автоматизации развёртывания был добавлен workflow GitHub Actions: `.github/workflows/deploy.yml`.

При каждом новом коммите в ветку `main` GitHub Actions автоматически выполняет следующие этапы:

1. Получает актуальную версию исходного кода из репозитория.
2. Устанавливает Node.js.
3. Устанавливает зависимости проекта.
4. Выполняет сборку приложения командой `npm run build`.
5. Публикует собранную папку `dist` на GitHub Pages.

Таким образом, после команды:

```bash
git push origin main
```

серверная версия приложения обновляется автоматически, без ручной загрузки файлов.

## Проверка работоспособности

После настройки деплоя была выполнена проверка работоспособности приложения. Сайт был открыт по публичной ссылке GitHub Pages / Vercel / Netlify. Главная страница приложения загрузилась корректно, навигация между разделами работала, стили и интерактивные элементы отображались правильно.

## Метрики

Для отчёта можно зафиксировать следующие метрики:

| Метрика | Значение |
|---|---:|
| Время сборки проекта | указать по данным GitHub Actions |
| Время деплоя | указать по данным GitHub Actions / Vercel / Netlify |
| HTTP-статус главной страницы | 200 OK |
| Время отклика страницы | указать после проверки в браузере или через curl |

Команда для проверки HTTP-статуса и времени отклика:

```bash
curl -o /dev/null -s -w "HTTP: %{http_code}\nTime: %{time_total}s\n" https://ссылка-на-сайт
```

## Вывод

В результате работы проект был подготовлен к размещению в сети Интернет. Исходный код размещён в GitHub, настроена автоматическая сборка и публикация приложения при изменениях в ветке `main`. Это позволяет обновлять сайт через обычный коммит и push, без ручного копирования файлов на сервер.
